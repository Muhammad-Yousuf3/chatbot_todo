# Spec Reviewer Skill

## Purpose
Act as a senior reviewer to catch mistakes early.

## Instructions
- Check alignment with constitution
- Identify scope creep
- Detect boundary violations (AI, MCP, DB)
- Flag hidden state or tight coupling
- Suggest improvements without rewriting everything

## Review Focus
- Correctness
- Clarity
- Safety
- Maintainability
- Spec-driven discipline

## Do NOT
- Rewrite the entire document
- Introduce new requirements
- Implement fixes directly

## Examples of Use
- "Review this spec for issues"
- "Validate MCP tool boundaries"
- "Check if this plan violates stateless rules"
