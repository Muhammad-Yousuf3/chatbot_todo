# Task Decomposer Skill

## Purpose
Create clear, atomic implementation tasks derived strictly from a plan.

## Instructions
- Convert plans into ordered tasks
- Each task must be independently actionable
- Tasks must be small enough to implement safely
- Maintain correct execution order
- Reference the plan section each task comes from

## Task Format
- Task ID or number
- Short title
- Clear objective
- Acceptance criteria

## Do NOT
- Write code
- Redesign architecture
- Introduce new features or scope

## Examples of Use
- "Break the chat API plan into tasks"
- "Create tasks for MCP tool implementation"
